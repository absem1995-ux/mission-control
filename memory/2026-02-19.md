## Critical Design Decisions Made
1. **Unified skill map over distributed configs:** Single atlas-config.json eliminates complexity, enables consistency across deployment models
2. **4-layer pipeline pattern:** Standardized input→create→distribute→learn for all agents (replicable to Astra, Sentinel, Quinn)
3. **Morgan as hiring manager:** COO role is team composition, not execution. Enables dynamic scaling per client.
4. **ZERO_ERRORS protocol:** Verification before output, certainty tracking—embedded in every agent
5. **Lessons-based learning:** 7-field format captures successes/failures, auto-propagates cross-agent via Morgan daily routine
6. **Telegram bots for inter-agent communication:** 1 bot per agent per client, no centralized service bottleneck
