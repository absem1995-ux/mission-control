## HOW MORGAN HIRES AGENTS

### Command Format

```
"Morgan, hire [agent-name] for [client-name]"
"Morgan, fire [agent-name] from [client-name]"
"Morgan, recommend a team for [client-name] with budget $[X]"
"Morgan, upgrade [client-name] to [tier]"
"Morgan, what's the cost if we add [agent-name]?"
```

### Hiring Process (Morgan)

1. **Client requests agent**
2. **Morgan checks:**
   - Is agent available? (below capacity limit)
   - Is client tier appropriate?
   - Is cost within budget?
3. **Morgan approves/denies:**
   - ✅ "Yes, hiring [agent]. Cost: $X/mo"
   - ❌ "No, [reason]. Alternative: [suggestion]"
4. **If approved:**
   - Spawn agent session (client-specific)
   - Create Telegram bot
   - Add to DEPLOYMENT.json
   - Activate agent's workspace
   - Notify all parties

### Firing Process (Morgan)

1. **Client requests agent removal**
2. **Morgan checks:**
   - Is agent active for this client?
   - Are there dependencies? (other agents relying on them)
3. **Morgan approves:**
   - "Confirmed, firing [agent]. Refund: $X"
4. **If approved:**
   - Kill agent session
   - Archive lessons to client workspace
   - Remove from DEPLOYMENT.json
   - Deactivate Telegram bot
   - Update DEPLOYMENT.json
   - Calculate refund (prorated)
